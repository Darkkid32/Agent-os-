# ADR template

Use this template when proposing a new architectural decision. File names
follow `NNN-short-kebab-case.md` (zero-padded) and live in `docs/adr/`.

## Status

One of `Proposed`, `Accepted`, `Superseded`, `Deprecated`.

## Context

- What is happening today?
- What forces are at play (technical, organizational, legal)?
- What constraints apply?

## Decision

State the decision in one sentence, then expand on the chosen approach.
Include the smallest possible surface change that satisfies the decision.

## Consequences

- What becomes easier because of this change?
- What becomes harder?
- List trade-offs explicitly.
- Flag follow-up work and any decisions this _decision_ now excludes.

## Alternatives Considered

What other options were considered, and why was this one picked instead?

## References

- Internal docs (`docs/...`).
- External papers, RFCS, products.
